##### **GTA V – QA Testing \& Bug Reporting Project**



###### **Project Overview:**

This project showcases my Quality Assurance (QA) testing work on Grand Theft Auto V E\&E (Story Mode \& Online)

The primary focus was to identify, document, and track in-game bugs using Jira in a structured and professional manner.



###### **The goal of this project is to demonstrate:**

* Real-world bug reporting skills
* Understanding of QA workflows
* Ability to reproduce and document issues clearly



###### **Testing Scope:**

The testing covered various aspects of GTA V gameplay, including:

* Character interactions
* Physics \& environmental behavior
* World assets and collision
* Mission-based scenarios
* Object interactions and mechanics



###### **Tools Used:**

* Jira – Bug tracking and reporting
* Microsoft Excel – Test case management
* GitHub – Project documentation and version control



###### **Bug Reporting Approach:**

Each bug was reported in a structured format using Jira, including:

* Summary – Short description of the issue
* Description – Detailed explanation
* Pre-Conditions – Required setup before testing
* Steps to Reproduce – Clear reproduction steps
* Expected Result – Intended behavior
* Actual Result – Observed behavior
* Severity – Impact level of the bug
* Reproducibility – Frequency of occurrence
